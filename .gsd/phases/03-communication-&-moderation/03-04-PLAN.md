<task type="auto">
  <name>Create supervisor conversation APIs</name>
  <files>src/app/api/supervisor/conversations/route.ts, src/app/api/supervisor/conversations/[matchId]/route.ts, src/app/api/supervisor/conversations/[matchId]/close/route.ts</files>
  <action>
  Create GET /api/supervisor/conversations:
  - Authenticate with auth(), verify role is SUPERVISOR
  - Get supervisor's assigned regions (supervisorRegions relation)
  - Query Match where status is ACTIVE or CLOSED
  - Filter: user1.regionId OR user2.regionId in supervisor's regions (cross-region support)
  - Include: user1, user2, messages (count only or last message)
  - Order by updatedAt DESC
  - Return {conversations: Match[]}
  
  Create GET /api/supervisor/conversations/[matchId]:
  - Authenticate with auth(), verify role is SUPERVISOR
  - Extract matchId from params
  - Verify match participants belong to supervisor's regions
  - Query match with all messages (include sender)
  - Return {match: Match, messages: Message[]}
  - Return 403 if supervisor not authorized for this region
  
  Create POST /api/supervisor/conversations/[matchId]/close:
  - Authenticate with auth(), verify role is SUPERVISOR
  - Extract matchId and {reason: string} from request
  - Verify match participants belong to supervisor's regions
  - Update Match: status = CLOSED, closedBy = supervisor ID, closedAt = now(), closedReason = reason
  - Create AuditLog entry: action = "CLOSE_CONVERSATION", target = matchId
  - Return 200 with updated match
  
  Follow patterns from src/app/api/supervisor/pending/route.ts for authentication and region checks.
  </action>
  <verify>curl GET /api/supervisor/conversations returns conversation list, POST close updates status</verify>
  <done>Supervisor APIs allow viewing conversations and closing chatrooms with audit logging</done>
</task>
